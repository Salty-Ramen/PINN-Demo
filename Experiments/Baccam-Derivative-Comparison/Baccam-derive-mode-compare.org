* Goals

Compare all four combinations of `LuxMLP{IC, D}`:

| IC axis      | Derivative axis  |
|--------------+------------------|
| SoftIC       | HandFD           |
| SoftIC       | ForwardDiffAD    |
| HardwiredIC  | HandFD           |
| HardwiredIC  | ForwardDiffAD    |

Three things are checked:

1. *Sanity at fixed parameters* — do both derivative methods agree on
   `dŷ/dt` for the same trained network, before any retraining?
2. *Training stability* — does each combination train without AD errors,
   and how does loss evolve?
3. *Final fit quality* — are the recovered ODE parameters and state
   trajectories comparable across methods?

Run on Baccam config B (10 timepoints × 12 mice).  Iteration counts are
reduced from the production sweep to keep the comparison tractable;
absolute MSE values will be higher than a fully-converged run, but the
*relative* ordering across methods is what matters.

function Baccam_model!(du, u, p, t)
    T, I, V = u
    du[1] = -p.β * T * V
    du[2] =  p.β * T * V - p.δ * I
    du[3] =  p.p * I     - p.c * V
end

true_params = (β = 2.7f-5, δ = 4.0f0, p = 1.2f-2, c = 3.0f0)
u0 = Float32[4.0f8, 0.0f0, 9.3f-2]
t_span = (0.0f0, 8.0f0)
N_fine = 2001
t_fine = collect(range(t_span[1], t_span[2], length = N_fine))

prob = ODEProblem(Baccam_model!, u0, t_span, true_params)
sol  = solve(prob, Tsit5(); saveat = t_fine)
sol_array = Array(sol)

rng = MersenneTwister(42)
noise_σ = 0.5f0
n_tp, n_mice = 10, 12

idx = round.(Int, range(1, length(t_fine), length = n_tp))
t_obs  = Float32.(t_fine[idx])
Y_true = Float32.(sol_array[:, idx])

ε = randn(rng, Float32, 3, n_tp, n_mice)
Y_noisy = Y_true .* exp.(noise_σ .* ε .- 0.5f0 * noise_σ^2)
Y_noisy = clamp.(Y_noisy, 0.0f0, Inf32)
Y_mean = dropdims(mean(Y_noisy; dims = 3); dims = 3)

println("Training data ready: $(n_tp) timepoints × $(n_mice) mice")

struct BaccamParams{T<:AbstractFloat}
    β::T; δ::T; c::T
end
BaccamParams(nt::ComponentVector) = BaccamParams(nt.β, nt.δ, nt.c)

function BaccamArchitectureRaw(y, g, p::BaccamParams)
    T, I, V = y[1,:], y[2,:], y[3,:]
    g_T, g_V = g[1,:], g[2,:]
    dT = g_T
    dI = p.β .* T .* V .- p.δ .* I
    dV = g_V .- p.c .* V
    return permutedims(hcat(dT, dI, dV))
end

build_state_mlp() = Lux.Chain(
    Lux.Dense(1, 8, tanh_fast),
    Lux.Dense(8, 6, tanh_fast),
    Lux.Dense(6, 3),
)

build_g_mlp() = Lux.Chain(
    Lux.Dense(1, 6, tanh_fast),
    Lux.Dense(6, 4, tanh_fast),
    Lux.Dense(4, 2),
)

state_transform = LogTransform(
    10f0,
    Float32[1f-6, 1f-6, 1f-6],

t_dense = permutedims(Float32.(range(minimum(t_obs), maximum(t_obs); length = 1000)))

* Sanity note

The training comparison below is itself the sanity check: if any
derivative method has wiring problems, it will surface immediately as

- A warning at Stage 2 start (e.g. "ForwardDiff within Zygote cannot
  track gradients" — indicates StatefulLuxLayer wiring isn't engaging
  Lux's nested-AD rule).
- NaN gradients during training.
- Vastly degraded final ode_mse compared to the FD baseline.

A separate "do the methods agree on dŷ/dt at fixed parameters" check
adds complexity (recovering trained Lux states for cross-method evaluation)
without much information beyond what the training run itself reveals.

* Training comparison — all 4 combinations

** Callback

Prints iteration count every 10 iters, with the stage tagged and the
wall-clock seconds-per-iteration over the last 10-iter window.  Stage
detection uses `ctx isa PINNCtxStage2` — same pattern used in the sweep
worker callback.  Output will be heavy (~7000 lines for 5k+10k iters)
but useful for confirming the AD path isn't silently stalling and for
spotting per-iteration cost differences between FD and AD live.

#+begin_src julia-vterm :results nil

# Closure over a Ref so we can mutate the previous-timestamp without
# making `callback_iter10` itself stateful at the global level.  Reset
# this between training runs so the first window of each new run isn't
# inflated by Julia's compilation latency carried over from the prior run.
let
    last_t = Ref(time())

    global function callback_iter10(state, l)
        state.iter % 10 == 0 || return false
        now = time()
        s_per_iter = (now - last_t[]) / 10
        last_t[] = now
        stage = state.p isa PINNCtxStage2 ? "S2" : "S1"
        @printf("[%s] iter = %6d  loss = %.4e  %.3f s/iter\n",
                stage, state.iter, l, s_per_iter)
        return false
    end

    global reset_callback_iter10!() = (last_t[] = time(); nothing)
end
#+end_src

#+begin_src julia-vterm

hp_train = HyperParams(0.0f0, 0.1f0, 4f-4, 0.5f0, 0.5f0)

# Per-combination iteration budget.  Reduced from production for tractability.
# 5k / 10k gives a fast first look — relative ordering of methods is what
# matters here, not absolute fit quality.  Bump these up once the comparison
# is trusted.
maxiters_s1 = 5_000
maxiters_s2 = 500

# All 4 combinations
combos = [
    ("SoftIC + HandFD",        LuxMLP(SoftIC(),      HandFD())),
    ("SoftIC + ForwardDiffAD", LuxMLP(SoftIC(),      ForwardDiffAD())),
    ("HardIC + HandFD",        LuxMLP(HardwiredIC(), HandFD())),
    ("HardIC + ForwardDiffAD", LuxMLP(HardwiredIC(), ForwardDiffAD())),
]

results = Dict{String, Any}()

# Run sequentially so AD errors (if any) don't get masked by thread fan-out
for (label, mode) in combos
    println("\n──────────────────────────────────────────────────────")
    println("  Training: $label")
    println("──────────────────────────────────────────────────────")

    stats = @timed begin
        try
            reset_callback_iter10!()
            train(
                FixedHyper(), data,
                BaccamArchitectureRaw, BaccamParams, hp_train;
                transform         = state_transform,
                arch_mode         = mode,
                seed              = 1,
                maxiters_stage1   = maxiters_s1,
                maxiters_stage2   = maxiters_s2,
                Opt_alg_stage1    = OptimizationOptimisers.Adam(1f-4),
                Opt_alg_stage2    = OptimizationOptimisers.Adam(1f-4),
                build_state_mlp   = build_state_mlp,
                build_g_mlp       = build_g_mlp,
                callback_function = callback_iter10,
            )
        catch e
            @warn "FAILED: $label" exception = (e, catch_backtrace())
            nothing
        end
    end

    results[label] = (res = stats.value, time_s = stats.time, mode = mode)

    if isnothing(stats.value)
        @printf("  [%s] FAILED in %.1fs\n", label, stats.time)
    else
        r = stats.value
        @printf("  [%s] wall=%.1fs  data_mse=%.3e  ode_mse=%.3e  β=%.2e  δ=%.3f  c=%.3f\n",
                label, stats.time,
                r.metrics.data_mse, r.metrics.ode_mse,
                r.params.ODE_par.β, r.params.ODE_par.δ, r.params.ODE_par.c)
    end
end

#+RESULTS:

* Notes on interpretation

When reading these results, what to look for:

1. *In the training comparison* — wall time per iteration should rank
   roughly: HandFD < ForwardDiffAD (per-step cost).  If AD wall time is
   a *lot* worse than FD (10×+), the StatefulLuxLayer wiring isn't
   engaging Lux's nested-AD rule — check Stage 2 logs for the "ForwardDiff
   within Zygote cannot track gradients" warning.  Final fit quality
   should be comparable across methods on the same IC.

2. *Soft IC vs Hardwired IC* — independently of derivative method,
   hardwired IC should give exact y(0) = y₀ and may converge faster in
   the early iterations; the IC penalty is one less competing gradient
   signal.  Final fit quality often comparable; the soft form has more
   flexibility but pays the IC-loss tax.

3. *NaN gradients* — if any combination NaNs out and recovers (the
   optimizer skips updates), the run completes but with degraded metrics.
   This was observed historically for HardIC + HandFD around iteration
   9000 and may resurface; if so, smaller learning rate or larger FD step
   are the typical fixes.
