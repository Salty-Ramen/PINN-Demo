* Background

This notebook tests the hypothesis that replacing the default MSE-with-data-std
data loss with a *proper Gaussian log-likelihood in transformed space*
improves convergence for the Baccam model.

** Data-generating process

The synthetic dataset uses multiplicative log-normal noise:

  y_obs = y_true · exp(σ·ε - σ²/2),    ε ~ N(0,1),  σ = 0.5

Taking logs:

  log y_obs = log y_true + σ·ε - σ²/2

so log y_obs is exactly Gaussian with constant variance σ². The noise is
*homoscedastic in log-space* by construction. There is no per-timepoint
or per-state variance structure to discover — it is the same σ everywhere.

** Why log-likelihood, not MSE-with-data-std

The default =loss_Data= divides residuals by =Y_train_std= (per-state std
of the training trajectory). This is the *signal* scale, not the *noise*
scale. The result is an MSE-shaped objective with a denominator chosen
for numerical convenience, not statistical correctness.

The Gaussian NLL in transformed coordinates is:

  -log p ∝ (1/2σ²) Σ (z_obs - ẑ)²

where z = log_b(y + s) is the transformed coordinate the network operates
in, σ is the *known noise std in z-space* (= 0.5), and the sum runs over
all observations (states × timepoints × mice). No ε_Data is needed — σ²
already plays that role and is determined by the noise model rather than
being a free hyperparameter.

** Per-timepoint vs global σ — homoscedasticity decision

The data is homoscedastic in log-space by construction, so a single global
σ is the correct model. Per-timepoint σ_k would add 10 free parameters per
state for no signal gain, and individual estimates have standard error
σ/√(2(n-1)) ≈ 0.11 with 12 mice — bouncing between 0.4 and 0.6 even when
the true σ is exactly 0.5. We use a single σ = 0.5 fixed.

For real data with possible heteroscedasticity, a per-state learned σ_i
(with =log σ_i= as a trainable parameter and the corresponding =N·log σ=
term in the likelihood) is the natural extension. We toggle this off here
because we know the truth.

** Using the full noisy dataset, not the means

The previous version collapsed 12 mice per timepoint into a single mean
trajectory. Under log-normal noise this is biased (Jensen's inequality:
log of the mean ≠ mean of the logs) and discards information (120 noisy
observations vs 10 means). The NLL rows train on all 120 observations
flattened; the default rows still train on means for backward comparison.

** Comparison matrix

|                                  | Soft IC | Hardwired IC |
|----------------------------------+---------+--------------|
| Default MSE + Log transform      |    A    |      B       |
| Gaussian NLL + Log transform     |    C    |      D       |

The transform is now identical across both rows: =LogTransform= maps the
multi-magnitude raw states into log10-coordinates where a plain linear-output
MLP works fine. The only difference between rows is the loss function and
the dataset shape (means vs full mouse data). This isolates the loss-function
effect cleanly.

** Loss-tracking callback

Same as the IC-ramp comparison — accumulates loss history with a continuous
iteration counter across Stage 1 and Stage 2 so the loss curves are plotted

#+begin_src julia-vterm :results nil

function make_loss_tracker(architecture, ode_param_ctor, maxiters_s1;
                           log_every::Int = 100,
                           data_loss_fn = loss_Data)
    hist = (
        iter     = Int[],
        data_mse = Float32[],
        ode_mse  = Float32[],
        stage    = Int[],
    )

    function cb(state, l)
        state.iter % log_every != 0 && return false

        ps  = state.u
        ctx = state.p

        is_s2 = ctx isa PINNCtxStage2
        global_iter = is_s2 ? maxiters_s1 + state.iter : state.iter

        # data_loss_fn is whatever loss is being tracked — MSE or NLL
        d_mse = data_loss_fn(ps, ctx)
        o_mse = is_s2 ? loss_ODE(ps, ctx, architecture, ode_param_ctor) : NaN32

        push!(hist.iter,     global_iter)
        push!(hist.data_mse, d_mse)
        push!(hist.ode_mse,  o_mse)
        push!(hist.stage,    is_s2 ? 2 : 1)

        return false
    end

    return cb, hist
end

** NaN/Inf diagnostic callback

Wraps an inner callback (e.g. =make_loss_tracker= or =callback_iter1000=)
with a one-shot diagnostic that fires the first time the loss or any
component goes non-finite. Probes the four mathematical danger zones
identified for the (loss, IC) configurations:

  - =extrema(ẑ)= on the dense grid: catches A.1 overflow (ẑ > 37) and
    A.2 underflow (ẑ < -37) which would make =b^ẑ= overflow or the
    chain-rule denominator =(y+s)·ln b= underflow to zero.
  - =extrema(NN_output)= on the dense grid: for hardwired IC, separates
    "the network is fine but t·NN amplifies it" (B.2) from "the network
    itself is producing extreme values."
  - =extrema(g_output)= on the dense grid: catches a runaway g-network
    driving the ODE residual.
  - Each loss component evaluated independently: tells you whether
    data, IC, or ODE is the source. A finite forward loss with NaN
    gradient indicates the indeterminate-form-in-chain-rule case.

The first non-finite event is printed under a thread lock so per-mode
prefixes don't interleave. Subsequent events from the same mode are
suppressed (one-shot) — once you know which probe fired first, that
identifies the case and further reports add noise.

Set =threshold_z = 30= rather than 37 so the warning fires *before*
overflow rather than after, giving a cleaner diagnosis (the iteration
where ẑ first crossed the danger threshold, not the iteration where
arithmetic broke).

#+begin_src julia-vterm :results nil

const NAN_DIAG_LOCK = ReentrantLock()

"""
    make_nan_diagnostic(label, architecture, ode_param_ctor, transform;
                        inner_cb, threshold_z, threshold_nn, threshold_g,
                        check_every) -> callback

Returns a callback that runs `inner_cb` first, then on the same
iteration evaluates per-config danger probes. Prints a one-shot
report the first time any probe trips, including:
  - which probe fired (ẑ, NN, g, or a NaN/Inf in a loss component)
  - the iteration number
  - the values of all probes at that iteration
  - which mathematical case (A.1/A.2/B.2/etc.) it most resembles

After firing once, the diagnostic stays silent for the rest of the run.
"""
function make_nan_diagnostic(label::String, architecture, ode_param_ctor, transform;
                             inner_cb = (s, l) -> false,
                             threshold_z::Real  = 30f0,
                             threshold_nn::Real = 30f0,
                             threshold_g::Real  = 1f10,
                             check_every::Int   = 25)
    fired = Ref(false)

    function cb(state, l)
        # Always run the inner callback first
        inner_cb(state, l)

        fired[] && return false
        state.iter % check_every != 0 && return false

        ps  = state.u
        ctx = state.p
        is_s2 = ctx isa PINNCtxStage2

        # ── Probes available in BOTH stages ──────────────────
        # ẑ on the training grid (the only grid Stage 1 has access to).
        # In Stage 2 we'd rather use t_dense (catches divergence
        # *between* observation timepoints), so override below.
        t_probe = is_s2 ? ctx.t_dense : ctx.t_train
        ẑ = ctx.predict_state(ps, t_probe)
        z_min, z_max = extrema(ẑ)
        z_extreme = max(abs(z_min), abs(z_max))

        loss_d = try loss_Data(ps, ctx) catch; NaN32 end
        loss_ic = try ctx.skip_ic_loss ? 0f0 : loss_IC(ps, ctx)
                  catch; NaN32 end

        # ── Stage 2 only probes (g-network, ODE residual) ────
        g_min = NaN32; g_max = NaN32; g_extreme = NaN32
        loss_o = NaN32

        if is_s2
            g = ctx.predict_g(ps, ctx.t_dense)
            g_min, g_max = extrema(g)
            g_extreme = max(abs(g_min), abs(g_max))
            loss_o = try loss_ODE(ps, ctx, architecture, ode_param_ctor)
                     catch; NaN32 end
        end

        # ── Trigger conditions ───────────────────────────────
        # In Stage 1, g and loss_o are NaN by construction (not present),
        # so don't count them toward `any_nan`.
        any_nan = !isfinite(l) || !isfinite(loss_d) || !isfinite(loss_ic) ||
                  !isfinite(z_extreme) ||
                  (is_s2 && (!isfinite(loss_o) || !isfinite(g_extreme)))
        z_huge = z_extreme > threshold_z
        g_huge = is_s2 && g_extreme > threshold_g

        if !(any_nan || z_huge || g_huge)
            return false
        end

        fired[] = true

        # ── Print under lock so per-thread output stays coherent ─────
        lock(NAN_DIAG_LOCK) do
            stage = is_s2 ? "S2" : "S1"
            ic_mode = ctx.skip_ic_loss ? "hardwired" : "soft"
            grid_name = is_s2 ? "t_dense" : "t_train"
            println("\n" * "="^72)
            @printf("[%s] DIAGNOSTIC FIRED at iter=%d (%s, %s IC)\n",
                    label, state.iter, stage, ic_mode)
            println("-"^72)
            @printf("  loss (full)      = %s\n",  isfinite(l) ? @sprintf("%.4e", l) : "$(l)")
            @printf("  loss_Data        = %s\n",  isfinite(loss_d) ? @sprintf("%.4e", loss_d) : "$(loss_d)")
            @printf("  loss_IC          = %s\n",  isfinite(loss_ic) ? @sprintf("%.4e", loss_ic) : "$(loss_ic)")
            if is_s2
                @printf("  loss_ODE         = %s\n",  isfinite(loss_o) ? @sprintf("%.4e", loss_o) : "$(loss_o)")
            end
            @printf("  ẑ on %-7s    = [%+.3f, %+.3f]   (|max| = %.2f)\n",
                    grid_name, z_min, z_max, z_extreme)
            if is_s2
                @printf("  g on t_dense     = [%+.3e, %+.3e]   (|max| = %.2e)\n",
                        g_min, g_max, g_extreme)
            end

            # Hardwired-IC: print raw NN output too (without t·NN amplification)
            if ctx.skip_ic_loss
                # Reconstruct raw NN output: ẑ = z₀ + t·NN ⇒ NN = (ẑ - z₀) / t
                # Skip t=0 column to avoid 0/0
                t_safe_idx = findall(t -> t > 0, vec(t_probe))
                if !isempty(t_safe_idx)
                    t_safe = t_probe[:, t_safe_idx]
                    ẑ_safe = ẑ[:, t_safe_idx]
                    z0_col = reshape(ctx.y0_obs, :, 1)
                    nn_out = (ẑ_safe .- z0_col) ./ t_safe
                    nn_min, nn_max = extrema(nn_out)
                    nn_extreme = max(abs(nn_min), abs(nn_max))
                    @printf("  NN output alone  = [%+.3f, %+.3f]   (|max| = %.2f)\n",
                            nn_min, nn_max, nn_extreme)
                    if nn_extreme < threshold_nn && z_huge
                        println("  → DIAGNOSIS: case B.2 — NN output is moderate but t·NN amplifies it")
                        println("    Fix: bound NN output (e.g. tanh final activation) or scale by 1/T")
                    end
                end
            end

            # Diagnose by elimination
            if is_s2 && !isfinite(loss_o) && isfinite(loss_d) && isfinite(loss_ic)
                println("  → DIAGNOSIS: ODE-residual term is the source")
                if z_extreme > 37
                    println("    Likely case A.1: ẑ overflowed → b^ẑ → Inf in transformed_rhs")
                elseif z_min < -37
                    println("    Likely case A.2: ẑ underflowed → chain-rule denom = b^ẑ·ln(b) → 0")
                elseif g_extreme > threshold_g
                    println("    Likely runaway g-network — its output dominates the ODE residual")
                end
            elseif !isfinite(loss_d) && (!is_s2 || !isfinite(loss_o))
                println("  → DIAGNOSIS: data term blew up — network output divergence")
                println("    In Stage 1, only the state network is being trained; check its init.")
            elseif !isfinite(loss_ic)
                println("  → DIAGNOSIS: IC loss source — soft IC with extreme ẑ(0)")
            elseif z_huge && !any_nan
                println("  → EARLY WARNING: ẑ exceeded safe range but loss is still finite")
                println("    Catch this BEFORE arithmetic breaks. Lower learning rate or check init.")
            end
            println("="^72 * "\n")
        end

        return false
    end

    return cb
end

#+RESULTS:

** Log-likelihood loss functions

These mirror =supervised_loss= and =unsupervised_loss= from Losses.jl but
swap =loss_Data= for =loss_Data_NLL_transformed=.  Everything else (IC
penalty, ODE residual, L1 regularisation, bound penalty, hyper penalty)
stays identical.

The =σ_NLL= constant is the known multiplicative noise std in log-space
(=bac_noise_σ= below). Since the network operates in log-coordinates, the
noise std applies directly to residuals between predictions and transformed
observations — no need for a numerical floor inside another log() call.

There is no ε_Data in the NLL composites: σ² already plays that role,
and adding ε_Data on top would be a redundant degree of freedom that
double-counts the noise model.

#+begin_src julia-vterm :results nil

const σ_NLL = 0.5f0     # multiplicative log-normal noise std (= noise std in log-space)

"""
    loss_Data_NLL_transformed(ps, ctx) -> Float32

Negative log-likelihood (up to θ-independent constants) under the
homoscedastic Gaussian noise model in *transformed* coordinates:

    -log p(z_obs | θ) ∝ (1 / 2σ²) Σ (z_obs - ẑ)²

The state predictor `ctx.predict_state` returns ẑ in transformed coords;
ctx.Y_train must already contain z_obs = forward_state(transform, y_obs)
in the same coords. The full noisy dataset (all mice flattened) lives
here, with t_train repeated to match.

This is the maximum-likelihood data-fit objective. No ε_Data weight is
applied because σ² already determines the term's magnitude. Returns the
*mean* of squared residuals divided by 2σ² so the loss has the same
dimensional scaling as the per-observation NLL contribution.
"""
function loss_Data_NLL_transformed(ps, ctx)
    ẑ = ctx.predict_state(ps, ctx.t_train)
    return Statistics.mean(abs2, vec(ẑ - ctx.Y_train)) / (2f0 * σ_NLL^2)
end

"""
    supervised_loss_NLL(ps, ctx, mode) -> Float32

Stage 1 NLL loss. Data term carries no ε_Data — σ² is the noise model.
The remaining ε's govern non-likelihood terms (L1, IC) that genuinely
need balancing relative to the likelihood scale.
"""
function supervised_loss_NLL(ps, ctx, mode::AbstractHyperMode)
    hp = resolve_epsilons(ps, ctx, mode)
    penalty = hyper_penalty(ps, mode)

    L = loss_Data_NLL_transformed(ps, ctx) +                  # σ-weighted internally
        l1_penalty_state(ps) / (hp.ϵ_L1_state^2 + 1f-6) +
        penalty

    if !ctx.skip_ic_loss
        L += loss_IC(ps, ctx) / (hp.ϵ_ic^2 + 1f-6)
    end

    return 0.5f0 * L
end


"""
    unsupervised_loss_NLL(ps, ctx, mode, architecture, ode_param_constructor) -> Float32

Stage 2 NLL loss. Data term σ-weighted internally. ODE residual is still
ε-weighted because it is not a likelihood — it is a soft constraint with
a separately-tunable strength.
"""
function unsupervised_loss_NLL(ps, ctx, mode::AbstractHyperMode,
                               architecture, ode_param_constructor)
    hp = resolve_epsilons(ps, ctx, mode)
    penalty = hyper_penalty(ps, mode)

    L = loss_Data_NLL_transformed(ps, ctx) +                  # σ-weighted internally
        l1_penalty_state(ps) / (hp.ϵ_L1_state^2 + 1f-6) +
        loss_ODE(ps, ctx, architecture, ode_param_constructor) /
            (hp.ϵ_ode^2 + 1f-6) +
        l1_penalty_g(ps) / (hp.ϵ_L1_g^2 + 1f-6) +
        bound_loss(ps, ctx) +
        penalty

    if !ctx.skip_ic_loss
        L += loss_IC(ps, ctx) / (hp.ϵ_ic^2 + 1f-6)
    end

    return 0.5f0 * L
end

function Baccam_model!(du, u, p, t)
    T, I, V = u
    du[1] = -p.β * T * V
    du[2] =  p.β * T * V - p.δ * I
    du[3] =  p.p * I     - p.c * V
end

struct BaccamParams{T<:AbstractFloat}
    β::T; δ::T; c::T
end
BaccamParams(nt::ComponentVector) = BaccamParams(nt.β, nt.δ, nt.c)

function BaccamArchitectureRaw(y, g, p::BaccamParams)
    T, I, V = y[1,:], y[2,:], y[3,:]
    g_T, g_V = g[1,:], g[2,:]
    dT = g_T
    dI = p.β .* T .* V .- p.δ .* I
    dV = g_V .- p.c .* V
    return permutedims(hcat(dT, dI, dV))
end

#+RESULTS:

* Training (parallel)

Run each cell on its own thread.  Start Julia with at least 4 threads
(=julia -t 4= or =JULIA_NUM_THREADS=4=).  BLAS threads are pinned to 1 per
task so optimisers don't contend.

#+begin_src julia-vterm

using Base.Threads

function callback_iter1000(state, l)
    state.iter % 1000 == 0 || return false
    stage = state.p isa PINNCtxStage2 ? "S2" : "S1"
    @printf("[%s] iter = %6d  loss = %.4e\n", stage, state.iter, l)
    return false
end

bac_tasks = Dict{String, Task}()
bac_loss_histories = Dict{String, Any}()

for (label, cfg) in bac_mode_configs
    tracker_cb, hist = make_loss_tracker(
        BaccamArchitectureRaw, BaccamParams, bac_maxiters_s1;
        data_loss_fn = cfg.track_data_loss,
    )
    bac_loss_histories[label] = hist

    # Chain: every iteration calls iter1000 (printout), then tracker
    # (history collection), then nan_diag (one-shot blow-up report).
    chained_cb(state, l) = (
        callback_iter1000(state, l);
        tracker_cb(state, l);
        false
    )

    nan_cb = make_nan_diagnostic(
        label, BaccamArchitectureRaw, BaccamParams, cfg.transform;
        inner_cb     = chained_cb,
        threshold_z  = 30f0,    # warn before b^ẑ overflows at 37
        threshold_g  = 1f10,    # g shouldn't exceed ~peak physics scale
        check_every  = 25,      # cheap; runs roughly every 25 iters
    )

    bac_tasks[label] = Threads.@spawn begin
        println("[Thread $(Threads.threadid())] Baccam — Starting $label")

        # Suppress Optimisers.jl's "NaN or Inf encountered" warning during
        # the train() call. Our nan_cb diagnostic catches and reports these
        # cases with full context (which loss term, which probe, which
        # iteration), so the built-in warning is redundant noise — and at
        # one warning per affected gradient step it can swamp the output.
        # `with_logger(NullLogger())` discards all `@warn`/`@info` calls
        # inside the wrapped block. `println` output (our diagnostic) is
        # unaffected because it doesn't go through the logging system.
        stats = with_logger(NullLogger()) do
            @timed train(
                FixedHyper(),
                cfg.data,
                BaccamArchitectureRaw,
                BaccamParams,
                cfg.hp;
                transform                  = cfg.transform,
                arch_mode                  = cfg.arch_mode,
                seed                       = 1,
                maxiters_stage1            = bac_maxiters_s1,
                maxiters_stage2            = bac_maxiters_s2,
                Opt_alg_stage1             = OptimizationOptimisers.Adam(bac_lr),
                Opt_alg_stage2             = OptimizationOptimisers.Adam(bac_lr),
                build_state_mlp            = cfg.build_state_mlp,
                build_g_mlp                = bac_build_g_mlp,
                callback_function          = nan_cb,
                default_supervised_loss    = cfg.sup_loss,
                default_unsupervised_loss  = cfg.unsup_loss,
            )
        end
        res = stats.value

        (res = res, time_s = stats.time, cfg = cfg)
    end
end

bac_results = Dict{String, Any}()
 for (label, t) in bac_tasks
    bac_results[label] = fetch(t)
    r = bac_results[label]
    @printf("\n  [Baccam %s] Wall: %.1fs  Data: %.4e  ODE: %.4e\n",
            label, r.time_s, r.res.metrics.data_mse, r.res.metrics.ode_mse)
