* Background

This notebook tests the hypothesis that *log-likelihood on the loss side* is
a better-conditioned alternative to *log-transform on the state side* for
the Baccam model.

The current default setup applies a composite =LogTransform ∘ ZScoreTransform=
to the state, then computes MSE in transformed coordinates.  The
LogTransform's chain-rule factor =1/(y + shift)= becomes large when =y= is
near its floor, which we conjecture amplifies near-IC noise in the ODE
residual loss and contributes to the convergence issues seen with hardwired
IC + log transform.

The proposed alternative: leave the state in raw coordinates and compute
the data-fit loss as the negative log-likelihood under the known
log-normal noise model.  Up to constants:

  NLL_data = (1 / 2σ²) Σ (log(y_data + ε) − log(ŷ + ε))²

with σ = 0.5 (known) and ε = 1e-6 (numerical floor to handle clamped zeros
in the noisy data).  The ODE residual loss stays as raw-coordinate MSE
weighted by =G_stage1_std=, so the chain-rule singularity is gone.

** Architectural challenge: signed magnitude

In raw coordinates the network must represent T (~4e8 decreasing), I
(~0 to ~1e8) and V (~0.07 to ~4e5).  A plain =Lux.Dense= with linear output
fed by =tanh= hidden layers cannot easily span these magnitudes.  An =exp=
final activation gives positive multi-magnitude output but cannot
represent T's decline (hardwired IC needs a signed contribution).

We use =sinh= as the final activation: signed, smooth, and sinh(x) for x∈[-20, 20]
spans the full range we need.  Per-state output scaling is applied as a fixed
post-multiplier so each state gets =sinh= in a useful range — see
=bac_build_state_mlp_nll= below.

** Comparison matrix

|                                | Soft IC | Hardwired IC |
|--------------------------------+---------+--------------|
| Default loss + Log+ZScore      |    A    |      B       |
| NLL loss + Identity (sinh out) |    C    |      D       |

Architecture is held fixed within each row.  The architecture difference
between rows is intrinsic to the (transform, loss) pair: log-state-transform
makes outputs O(1) so plain linear works; raw-state forces sinh + scaling.
This is a confounded variable but unavoidable — there is no architecture
that works equally well in both regimes.

** Loss-tracking callback

Same as the IC-ramp comparison — accumulates loss history with a continuous
iteration counter across Stage 1 and Stage 2 so the loss curves are plotted

#+begin_src julia-vterm :results nil

function make_loss_tracker(architecture, ode_param_ctor, maxiters_s1;
                           log_every::Int = 100,
                           data_loss_fn = loss_Data)
    hist = (
        iter     = Int[],
        data_mse = Float32[],
        ode_mse  = Float32[],
        stage    = Int[],
    )

    function cb(state, l)
        state.iter % log_every != 0 && return false

        ps  = state.u
        ctx = state.p

        is_s2 = ctx isa PINNCtxStage2
        global_iter = is_s2 ? maxiters_s1 + state.iter : state.iter

        # data_loss_fn is whatever loss is being tracked — MSE or NLL
        d_mse = data_loss_fn(ps, ctx)
        o_mse = is_s2 ? loss_ODE(ps, ctx, architecture, ode_param_ctor) : NaN32

        push!(hist.iter,     global_iter)
        push!(hist.data_mse, d_mse)
        push!(hist.ode_mse,  o_mse)
        push!(hist.stage,    is_s2 ? 2 : 1)

        return false
    end

    return cb, hist
end

#+RESULTS:

** Log-likelihood loss functions

These mirror =supervised_loss= and =unsupervised_loss= from Losses.jl but
swap =loss_Data= for =loss_Data_NLL=.  Everything else (IC penalty, ODE
residual, L1 regularisation, bound penalty, hyper penalty) stays
identical.

The =σ_NLL= constant is the known multiplicative noise level from the data
generator (=bac_noise_σ= below).  =NLL_FLOOR= is a numerical floor inside
the log to handle observations that were clamped to 0.

#+begin_src julia-vterm :results nil

const σ_NLL    = 0.5f0     # multiplicative log-normal noise std
const NLL_FLOOR = 1f-6     # floor inside log() for clamped zeros

"""
    loss_Data_NLL(ps, ctx) -> Float32

Negative log-likelihood (up to constants) under multiplicative log-normal
noise.  Compares predictions and observations in *raw* coordinates with a
log applied inside the squared difference.

Operates on whatever ctx.Y_train contains; ctx.predict_state output must be
in raw coordinates for this to be meaningful, so this loss should only be
used with IdentityTransform.
"""
function loss_Data_NLL(ps, ctx)
    ŷ = ctx.predict_state(ps, ctx.t_train)
    log_ŷ = log.(ŷ .+ NLL_FLOOR)
    log_y = log.(ctx.Y_train .+ NLL_FLOOR)
    return Statistics.mean(abs2, vec(log_ŷ - log_y)) / (2f0 * σ_NLL^2)
end

# Stage 1 (supervised) — replaces loss_Data with loss_Data_NLL
function supervised_loss_NLL(ps, ctx, mode::AbstractHyperMode)
    hp = resolve_epsilons(ps, ctx, mode)
    penalty = hyper_penalty(ps, mode)

    L = loss_Data_NLL(ps, ctx) / (hp.ϵ_Data^2 + 1f-6) +
        l1_penalty_state(ps)   / (hp.ϵ_L1_state^2 + 1f-6) +
        penalty

    if !ctx.skip_ic_loss
        L += loss_IC(ps, ctx) / (hp.ϵ_ic^2 + 1f-6)
    end

    return 0.5f0 * L
end

# Stage 2 (unsupervised / ODE-regularized)
function unsupervised_loss_NLL(ps, ctx, mode::AbstractHyperMode,
                               architecture, ode_param_constructor)
    hp = resolve_epsilons(ps, ctx, mode)
    penalty = hyper_penalty(ps, mode)

    L = loss_Data_NLL(ps, ctx)    / (hp.ϵ_Data^2 + 1f-6) +
        l1_penalty_state(ps)      / (hp.ϵ_L1_state^2 + 1f-6) +
        loss_ODE(ps, ctx, architecture, ode_param_constructor) /
            (hp.ϵ_ode^2 + 1f-6) +
        l1_penalty_g(ps)          / (hp.ϵ_L1_g^2 + 1f-6) +
        bound_loss(ps, ctx) +
        penalty

    if !ctx.skip_ic_loss
        L += loss_IC(ps, ctx) / (hp.ϵ_ic^2 + 1f-6)
    end

    return 0.5f0 * L
end

function Baccam_model!(du, u, p, t)
    T, I, V = u
    du[1] = -p.β * T * V
    du[2] =  p.β * T * V - p.δ * I
    du[3] =  p.p * I     - p.c * V
end

#+begin_src julia-vterm :results nil

struct BaccamParams{T<:AbstractFloat}
    β::T; δ::T; c::T
end
BaccamParams(nt) = BaccamParams(nt.β, nt.δ, nt.c)

function BaccamArchitectureRaw(y, g, p::BaccamParams)
    T, I, V = y[1,:], y[2,:], y[3,:]
    g_T, g_V = g[1,:], g[2,:]
    dT = g_T
    dI = p.β .* T .* V .- p.δ .* I
    dV = g_V .- p.c .* V
    return permutedims(hcat(dT, dI, dV))
end

#+RESULTS:

** State MLP — NLL (sinh output, per-state scale, used with identity transform)

The =sinh= activation gives signed multi-magnitude output (sinh(20) ≈ 2.4e8).
Per-state output scaling lets each state get sinh in a useful range without
the network having to learn extreme weight magnitudes.  Scales are
=Y_TRUE_SCALE= which corresponds to roughly the peak magnitude of each
true state — chosen from the analytic Baccam solution, not learned.

The trailing =Lux.WrappedFunction= multiplies the sinh output element-wise
by the scale vector.  Since this is a fixed (non-trainable) operation it
adds no parameters; it just re-scales the output to be in a learnable range.

#+begin_src julia-vterm :results nil

# Per-state magnitude scale — set from the true peak of each state.
# In a real application you'd derive this from the data:
# something like `vec(maximum(abs.(Y_train); dims=2))`.
const BAC_Y_SCALE = Float32[4f8, 1f8, 5f5]   # T peak, I peak, V peak

# The wrapper applies sinh and scales row-wise.  Defined as a plain function
# so it's StatefulLuxLayer-compatible.
function sinh_scale(x::AbstractMatrix)
    return BAC_Y_SCALE .* sinh.(x)
end

bac_build_state_mlp_nll() = Lux.Chain(
    Lux.Dense(1, 8, tanh_fast),
    Lux.Dense(8, 6, tanh_fast),
    Lux.Dense(6, 3),                      # raw signed output
    Lux.WrappedFunction(sinh_scale),      # final per-state scaling
)
#+end_src

* Transforms and data packing

** Default row — Log + ZScore composite

The default loss path uses the existing transform stack: log on the
states (so peak magnitudes become ~9, ~8, ~5.5), then z-score on the
log-coordinates so each row is roughly N(0,1).

#+begin_src julia-vterm :results nil

# Log on the raw states first
bac_log_tr = LogTransform(
    10f0,
    Float32[1f-6, 1f-6, 1f-6],
    Bool[true, true, true],
)

# Compute per-row mean and std of the LOG-transformed mean data so the
# z-score brings the network outputs to ~N(0,1)
bac_Y_log = forward_state(bac_log_tr, bac_Y_mean)
bac_zscore_μ = vec(mean(bac_Y_log; dims = 2))
bac_zscore_σ = vec(std(bac_Y_log; dims = 2))
bac_zscore_tr = ZScoreTransform(bac_zscore_μ, bac_zscore_σ)

# Composite transform: state → log → zscore
bac_transform_default = ComposedTransform(bac_log_tr, bac_zscore_tr)
