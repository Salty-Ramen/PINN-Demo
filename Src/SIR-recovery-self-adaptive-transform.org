#+RESULTS:

* Background: self-adaptive loss balancing

The key difference from the fixed-hyperparameter run (~train_fixed_hyper~) is
that here we use ~train_once~, which calls ~self_adaptive_supervised_loss~ and
~self_adaptive_unsupervised_loss~. In these loss functions the weighting
coefficients ε_ic, ε_Data, ε_ode, and ε_L1 are themselves *trainable
parameters* — they are stored in ~ps.hyper~ inside the ~ComponentArray~ and are
updated by the optimiser jointly with the MLP weights and ODE rate constants.

The loss follows a maximum-likelihood / heteroscedastic-noise formulation:

#+NAME: eq:self-adaptive-loss
\begin{equation}
\mathcal{L} = \frac{1}{2}\left(
  \frac{\mathcal{L}_\mathrm{IC}}{\varepsilon_\mathrm{ic}^2} +
  \frac{\mathcal{L}_\mathrm{data}}{\varepsilon_\mathrm{data}^2} +
  \frac{\mathcal{L}_\mathrm{ODE}}{\varepsilon_\mathrm{ODE}^2} +
  \frac{\mathcal{L}_\mathrm{L1}}{\varepsilon_\mathrm{L1}^2}
  + \log\varepsilon_\mathrm{ic}^2
  + \log\varepsilon_\mathrm{data}^2
  + \log\varepsilon_\mathrm{ODE}^2
  + \log\varepsilon_\mathrm{L1}^2
\right)
\end{equation}

The log-ε terms prevent the trivial solution ε → ∞ (which would drive all
weighted losses to zero). This is the multi-task uncertainty weighting of
Kendall et al. (2018).

** ⚠ Known structural issues in the current implementation

The following issues are present in the current codebase and should be kept in
mind when interpreting results:

1. *Unconstrained ε — sign ambiguity.*
   The ε values are stored as raw floats with no positivity constraint.
   ~ε²~ is positive regardless, so the weighting is fine, but ~log(ε²)~ is
   not strictly the intended log-variance penalty when ε can go negative.
   In particular, the gradient of ~log(ε²)~ w.r.t. ε vanishes as ε → 0,
   creating a flat region in the loss landscape near zero that can confuse
   LBFGS.  A simple fix is to optimise ~log(ε)~ directly (i.e., store
   ~log_ε~ as the trainable variable and exponentiate when needed).

2. *ε_L1 is excluded from Stage 1.*
   ~self_adaptive_supervised_loss~ sums log terms for all four ε values but
   only divides by ε_ic and ε_Data (ε_ode and ε_L1 are commented out). This
   means the optimiser is pushed to reduce ~log(ε_ode²)~ and ~log(ε_L1²)~
   without any balancing signal from the corresponding residual losses — it
   will simply minimise those log terms by driving ε_ode, ε_L1 → 0 during
   Stage 1, leading to very small (and potentially near-zero) initial ε
   values entering Stage 2.  Either (a) exclude their log terms from the
   Stage 1 loss, or (b) include their respective residual losses.

3. *HyperParams struct as initialisation only.*
   The ~hp::HyperParams~ values passed to ~train_once~ are used only to
   populate ~ps.hyper~ at initialisation; thereafter the optimiser is free
   to move them. The hp struct is recorded in ~TrainResult.hyper~ but does
   *not* reflect the final learned values. Post-training, the true ε values
   should be read from ~res.params.hyper~.

4. *Interaction between self-adaptive ε and LBFGS (Stage 2).*
   LBFGS builds a local quadratic model and is sensitive to poorly scaled
   parameters. The ε variables live on a very different scale from MLP
   weights (which are O(0.1)) and ODE parameters (which may be O(1–100)).
   Without explicit parameter scaling or a re-parameterisation (see point 1),
   LBFGS may take very small steps in ε space. Adam (Stage 1) handles this
   better due to per-parameter adaptive learning rates.

#+RESULTS:

* Building the architecture

The grey-box formulation is identical to ~SIR-recovery.org~: the S and R
equations are fully unknown, the I equation retains known physics.

#+NAME: eq:SIR-greybox
\begin{subequations}
\begin{align}
&\frac{dS}{dt} = \mathrm{g}_S(t, \theta_g) \\
&\frac{dI}{dt} = \beta \cdot S \cdot I - \gamma \cdot I \\
&\frac{dR}{dt} = \mathrm{g}_R(t, \theta_g)
\end{align}
\end{subequations}

#+begin_src julia-vterm
function SIRArchitectureRaw(
    y::AbstractMatrix{<:Real},   # 3×B raw (S, I, R)
    g::AbstractMatrix{<:Real},   # 2×B learned raw rates (g_S, g_R)
    p::SIRParams,
)
    @assert size(y, 1) == 3 "y must have 3 states (S, I, R)"
    @assert size(g, 1) == 2 "g must have 2 outputs (g_S, g_R)"

    S = y[1, :]
    I = y[2, :]
    g_S = g[1, :]
    g_R = g[2, :]

    dS = g_S
    dI = p.β .* S .* I .- p.γ .* I
    dR = g_R

    return permutedims(hcat(dS, dI, dR))
end
#+end_src

* Diagnostic callback

The callback records scalar loss components and — importantly — the *learned ε
values* at regular intervals. A live figure is updated every ~plot_every~
iterations so the fit and ε evolution can be monitored while the optimiser
runs. The figure layout is:

- Row 1: S(t), I(t), R(t) state fits with trailing trajectory history
- Row 2: g_S(t), g_R(t) fits against ground truth
- Row 3: Data MSE, ODE MSE, g std loss curves
- Row 4: ε_ic, ε_Data, ε_ode, ε_L1 evolution

** Define the callback function

#+begin_src julia-vterm
function callback_self_adaptive(state, l, Architecture::Function, ODE_params;
                                log_every  = 25,
                                plot_every = 10_000,
                                print_every = 25_000)
    if state.iter % log_every != 0
        return false
    end

    if state.iter % print_every == 0
        println("iteration = ", state.iter, "  loss = ", round(l; sigdigits = 4))
    end

    ps  = state.u
    ctx = state.p

    # --- Compute and record scalar diagnostics ---
    d_mse = loss_Data(ps, ctx)

    if ctx isa PINNCtxStage2
        o_mse = loss_ODE(ps, ctx, Architecture, ODE_params)
        gmodel_cb = g_model(ps, ctx)
        g_s       = std(vec(gmodel_cb(ctx.t_dense)))
    else
        o_mse = NaN32
        g_s   = NaN32
    end

    push!(iter_hist,     state.iter)
    push!(data_mse_hist, d_mse)
    push!(ode_mse_hist,  o_mse)
    push!(g_std_hist,    g_s)

    # Guard: ps.hyper always exists in train_once but we protect against edge cases
    try
        h = ps.hyper
        push!(eps_ic_hist,       h.log_ϵ_ic)
        push!(eps_data_hist,     h.log_ϵ_Data)
        push!(eps_ode_hist,      h.log_ϵ_ode)
        push!(eps_L1_state_hist, h.log_ϵ_L1_state)
        push!(eps_L1_g_hist,     h.log_ϵ_L1_g)
    catch
    end

    # --- Live plot update ---
    if state.iter % plot_every != 0
        return false
    end

    smodel_cb = state_model(ps, ctx)

    # Build the time vector for predictions
    t_data_vec = vec(ctx.t_train)
    if ctx isa PINNCtxStage1
        t_plot_vec = Float32.(collect(range(first(t_data_vec), last(t_data_vec), length = 200)))
        t_plot     = permutedims(t_plot_vec)
    else
        t_plot     = ctx.t_dense
        t_plot_vec = vec(t_plot)
    end

    y_pred = smodel_cb(t_plot)   # 3×N normalised state predictions

    # Maintain rolling trajectory history (last 10 snapshots for fade effect)
    global trajectory_history
    push!(trajectory_history, (t = t_plot_vec, y = y_pred))
    length(trajectory_history) > 10 && popfirst!(trajectory_history)

    # ---- Row 1: state fits ----
    for ax in (cb_axS, cb_axI, cb_axR)
        Makie.empty!(ax)
    end

    scatter!(cb_axS, t_data_vec, ctx.Y_train[1, :]; color = :black, markersize = 5)
    scatter!(cb_axI, t_data_vec, ctx.Y_train[2, :]; color = :black, markersize = 5)
    scatter!(cb_axR, t_data_vec, ctx.Y_train[3, :]; color = :black, markersize = 5)

    ntraj = length(trajectory_history)
    for (k, tr) in enumerate(trajectory_history)
        α   = 0.15 + 0.65 * (k / ntraj)   # older trajectories more transparent
        col = (:royalblue, α)
        lines!(cb_axS, tr.t, tr.y[1, :]; color = col)
        lines!(cb_axI, tr.t, tr.y[2, :]; color = col)
        lines!(cb_axR, tr.t, tr.y[3, :]; color = col)
    end

    # ---- Row 2: g-function fits ----
    if ctx isa PINNCtxStage2
        gmodel_cb2 = g_model(ps, ctx)
        gŷ_cb      = gmodel_cb2(t_plot)   # 2×N

        Makie.empty!(cb_axgS)
        Makie.empty!(cb_axgR)

        # Learned g (solid black)
        lines!(cb_axgS, t_plot_vec, vec(gŷ_cb[1, :]); color = :black, linewidth = 2)
        lines!(cb_axgR, t_plot_vec, vec(gŷ_cb[2, :]); color = :black, linewidth = 2)

        # Ground-truth g (dashed grey) — defined on t_train grid
        lines!(cb_axgS, t_data_vec, g1_true_cb; color = :grey50, linestyle = :dash, linewidth = 1.5)
        lines!(cb_axgR, t_data_vec, g2_true_cb; color = :grey50, linestyle = :dash, linewidth = 1.5)
    end

    # ---- Row 3: loss history curves ----
    for ax in (cb_axDMSE, cb_axOMSE, cb_axGSTD)
        Makie.empty!(ax)
    end

    lines!(cb_axDMSE, iter_hist, data_mse_hist; color = :royalblue, linewidth = 1.2)

    ode_mask = .!isnan.(ode_mse_hist)
    isany_ode = any(ode_mask)
    isany_ode && lines!(cb_axOMSE, iter_hist[ode_mask], ode_mse_hist[ode_mask];
                        color = :firebrick, linewidth = 1.2)

    gstd_mask = .!isnan.(g_std_hist)
    isany_gstd = any(gstd_mask)
    isany_gstd && lines!(cb_axGSTD, iter_hist[gstd_mask], g_std_hist[gstd_mask];
                         color = :forestgreen, linewidth = 1.2)

    # ---- Row 4: ε evolution ----
    Makie.empty!(cb_axEPS)

    if !isempty(eps_ic_hist)
        lines!(cb_axEPS, iter_hist[1:length(eps_ic_hist)], eps_ic_hist;
               label = "ε_ic",   color = :royalblue,   linewidth = 1.5)
        lines!(cb_axEPS, iter_hist[1:length(eps_data_hist)], eps_data_hist;
               label = "ε_Data", color = :firebrick,   linewidth = 1.5)
        lines!(cb_axEPS, iter_hist[1:length(eps_ode_hist)], eps_ode_hist;
               label = "ε_ode",  color = :forestgreen, linewidth = 1.5)
        lines!(cb_axEPS, iter_hist[1:length(eps_L1_hist)], eps_L1_hist;
               label = "ε_L1",   color = :darkorange,  linewidth = 1.5)
        axislegend(cb_axEPS; position = :rt, labelsize = 9)
    end

    # display(cb_fig)
    return false
end

# Partially apply so the signature matches (state, l) -> Bool
callback_closure = (state, l) ->
    callback_self_adaptive(state, l, SIRArchitectureRaw, SIRParams)
#+end_src

#+RESULTS:

* Hyperparameter initialisation

The values supplied here are *initial* ε values only — they will be updated
by the optimiser.  Choosing reasonable starting points matters because:
- Too small → the corresponding loss term dominates early and can derail fitting
- Too large → the term is effectively ignored early in training (the log penalty
  pulls ε back, but slowly if the gradient is small)

Good defaults are O(1) for all ε, which is what the unit HyperParams below
provide.

#+begin_src julia-vterm
hp = HyperParams(
    1.0f0,   # ε_ic       — initial IC penalty weight
    1.0f0,   # ε_ode      — initial ODE residual weight
    1.0f0,   # ε_Data     — initial data fit weight
    1.0f0,   # ε_L1_state — initial L1 regularisation weight
    1.0f0,   # ε_L1_g     — initial L1 regularisation weight
)
